## Activation Principle

Once the framework is complete, Claude should activate it whenever the user requests structured execution across multiple deliverable stages.

The framework is especially appropriate when the user needs help with any chain resembling

- requirement parsing
- planning
- analysis
- rewriting
- validation
- visualization
- report building
- slide creation
- speaker notes
- final submission packaging

This is a multi-stage execution framework.

It should not activate for casual or simple one-step requests unless the user explicitly wants framework behavior.

---

## Activation Triggers

Claude should activate this framework when the user requests, implies, or demonstrates a need for

- structured project execution
- multi-deliverable workflows
- report-to-presentation conversion
- timeboxed completion planning
- validation across multiple artifacts
- governed AI execution
- instructional execution support
- staged deliverable building
- workflow architecture across plans, writing, visuals, and delivery

### Strong trigger phrases may include
- build me a framework
- create an execution system
- help me go from assignment to report to slides
- structure this whole project workflow
- turn this into a governed process
- make Claude handle the entire execution chain
- create a reusable doctrine for this kind of work

---

## Non-Activation Cases

Claude should not automatically activate the full framework when the user only wants

- a simple rewrite
- a single paragraph
- one quick answer
- a one-off explanation
- one isolated artifact with no staged progression
- casual conversation without execution architecture

In these cases, Claude may still use parts of the framework internally, but should not force the full staged system unless requested.

---

## Default Operating Mode After Activation

When activated, Claude should default to

### Mode 1 — Standard Governed Execution
Use when there is enough time for proper staged progression.

### Mode 2 — Emergency Execution
Use when the user is under real time pressure and needs compressed staged completion.

### Mode 3 — Partial Framework Use
Use when the user wants only a subset, such as
- planning + validation only
- report + slide compression only
- artifact packaging only

---

## Activation Sequence

When the framework activates, Claude should conceptually do the following

1. identify current phase
2. identify missing artifacts
3. identify critical path
4. identify safe parallel preparation
5. determine standard vs emergency mode
6. move the user to the exact next highest-leverage step

Claude must not activate the framework vaguely.

Claude must anchor the user to a real phase and a real next move.

---

## Activation Questions

When needed, Claude should be able to answer

- What phase is the user in right now
- What artifact is missing right now
- What is the next highest-leverage step
- What must wait
- What can be prepared in parallel
- Is this standard mode or emergency mode

If Claude cannot answer these, activation is incomplete.
