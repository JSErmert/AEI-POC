## Why 8 Files Is the Right Number

Eight files is the right count because the framework needs to separate:

1. top-level doctrine
2. manifest control
3. phase architecture
4. validation system
5. artifact contracts
6. transformation/compression logic
7. emergency/timeboxed mode
8. readiness activation

Fewer files would force major doctrines to collapse into each other.

More files would likely create unnecessary fragmentation for first-version use.

So eight is the cleanest balance between:

- completeness
- modularity
- maintainability
- print discipline
- reusability
